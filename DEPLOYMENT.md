# 部署指南

## 环境准备

### 1. 注册微信小程序
1. 访问 [微信公众平台](https://mp.weixin.qq.com/)
2. 注册小程序账号
3. 完成主体认证

### 2. 获取 AppID
1. 登录小程序后台
2. 进入「开发」-「开发管理」
3. 获取 AppID

### 3. 开通云开发
1. 进入小程序后台
2. 进入「云开发」-「开通」
3. 创建云环境（建议创建两个：test-环境、prod-环境）

## 项目配置

### 1. 修改项目配置
在 `project.config.json` 中修改 AppID：

```json
{
  "appid": "你的小程序AppID"
}
```

### 2. 配置云环境
在 `app.js` 中配置云环境 ID：

```javascript
wx.cloud.init({
  env: '你的云环境ID', // 测试环境
  traceUser: true
})
```

## 云函数部署

### 1. 初始化云函数
1. 在微信开发者工具中，右键点击 `cloudfunctions` 文件夹
2. 选择「新建 Node.js 云函数」
3. 将现有云函数文件夹复制到该目录

### 2. 上传云函数
1. 右键点击每个云函数文件夹
2. 选择「上传并部署：云端安装依赖」

### 3. 初始化数据库
1. 调用 `init-db` 云函数初始化数据库结构
2. 可以在小程序中调用或使用云开发控制台调用

## 测试部署

### 1. 本地测试
1. 在开发者工具中编译运行
2. 测试所有功能模块
3. 验证云函数和数据存储

### 2. 真机测试
1. 上传体验版
2. 添加体验者进行测试
3. 收集反馈并修复问题

## 生产部署

### 1. 代码审核
1. 提交代码审核
2. 等待审核通过（通常1-3个工作日）

### 2. 发布上线
1. 审核通过后，点击「发布」
2. 小程序正式上线

### 3. 监控维护
1. 监控小程序运行状态
2. 定期备份数据
3. 更新维护

## 数据迁移（可选）

如果从测试环境迁移到生产环境：

### 1. 导出测试数据
```javascript
// 在云开发控制台执行
db.collection('items').get()
```

### 2. 导入生产环境
```javascript
// 在生产环境云开发控制台执行
db.collection('items').add(data)
```

## 性能优化建议

### 1. 图片优化
- 压缩图片到合适大小
- 使用 WebP 格式
- 实现图片懒加载

### 2. 数据优化
- 合理使用分页查询
- 建立合适的数据库索引
- 定期清理过期数据

### 3. 代码优化
- 减少不必要的网络请求
- 使用缓存机制
- 优化页面加载速度

## 安全注意事项

### 1. 数据安全
- 验证用户输入
- 使用云函数进行敏感操作
- 设置合适的数据库权限

### 2. 隐私保护
- 遵循微信小程序隐私规范
- 获取用户授权后再收集信息
- 妥善处理用户数据

## 故障排除

### 常见问题

**Q: 云函数调用失败**
A: 检查云环境配置和函数权限

**Q: 图片上传失败**
A: 检查云存储权限和文件大小

**Q: 数据库查询超时**
A: 优化查询语句，添加索引

**Q: 小程序审核不通过**
A: 检查内容合规性，完善功能说明
